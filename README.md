Лабораторная работа №4 — Unit тесты

Цели и задачи тестирования

Цель тестирования – проверить корректность работы модуля 'rectangle.py', вычисляющего площадь и периметр прямоугольника по заданным сторонам.

Основные задачи:
- убедиться в работоспособности функций 'area(a, b)' и 'perimeter(a, b)' на типичных и граничных наборах данных;
- подтвердить совпадение результатов ручного и автоматического тестирования.

Описание тестируемого продукта

Тестируемый продукт — модуль на языке Python 'rectangle.py', реализующий операции над прямоугольником.

Функциональность:
- 'area(a, b)' — возвращает площадь прямоугольника по двум сторонам 'a' и 'b';
- 'perimeter(a, b)' — возвращает периметр прямоугольника по сторонам 'a' и 'b';
- при запуске файла как программы осуществляется ввод значений сторон с клавиатуры и вывод площади и периметра в консоль.

Требования к продукту:
- корректно вычислять площадь и периметр прямоугольника при корректных числовых значениях сторон;
- корректно обрабатывать случай, когда одна из сторон равна нулю (площадь должна быть равна нулю).

Область тестирования

В рамках данной лабораторной работы тестированию подлежат:

Функция 'area(a, b)' при следующих наборах входных данных:
- 'a = 10', 'b = 0' — граничный случай;
- 'a = 10', 'b = 5' — типичный случай;
- 'a = 10', 'b = 10' — квадрат.

Функция 'perimeter(a, b)' при следующих наборах входных данных:
- 'a = 10', 'b = 5';
- 'a = 10', 'b = 10'.

Также проверяется консольный ввод и вывод при ручном запуске программы.

Случаи с отрицательными сторонами, некорректным вводом (строки, пустой ввод и т.п.) и экстремально большими числами в данной работе не рассматриваются.

Стратегия тестирования

Используется комбинированный подход: ручное тестирование и автоматическое unit-тестирование.

Ручное тестирование

Модуль 'rectangle.py' запускается как обычная программа командой:

python rectangle.py
